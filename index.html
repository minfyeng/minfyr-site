The commands that run the factory

Each stage is one slash command. Most auto-chain into the next.

Customer delivery pipeline Brief to production · Jira + Confluence · UX sign-off · multi-env AWS infra
/minfyr:planPLAN

Plan

Drives the full pipeline from a SOW or brief — auto-chains UX, stack, tech-spec and tickets. Output: signed backlog ready to build.

/minfyr:ux-builderUX

UX design

Builds UX from scratch — positioning brief → theme → wireframes → component contracts → HTML mockups deployed for customer review.

/minfyr:ux-approveUX

UX sign-off

Captures customer UX approval, hash-seals the API contract, and auto-chains to stack-picker → tech-spec → ticket generation.

/minfyr:stack-pickerSTACK

Stack

Picks framework, DB, auth, compute, IaC, repo layout and AI provider. Re-run with --reopen to revise a single decision.

/minfyr:tech-spec-writerSPEC

Tech spec

Writes DB models, API contract, per-env AWS infra, VPC, NFRs and the security model. Auto-chains to ticket generation.

/minfyr:ticket-generatorBACKLOG

Backlog

Generates the topologically-sorted backlog with a dependency graph and per-feature slice caches from the product and tech specs.

/minfyr:sprint-planSPRINT

Sprint plan

Buckets the sorted backlog into 2-week sprints between a start and end date, reserving final test and deploy weeks.

/minfyr:tasks-to-trackerJIRA

Jira push

Pushes the backlog to Jira — creates Epics, stories and tasks. Idempotent: re-runs update existing issues without duplicating.

/minfyr:implement-featureBUILD

Build

First run auto-scaffolds FE, BE, infra and design repos (design tokens, locked contract, dual-CI and cloud deploy baked in). Then implements each feature slice with TDD on a Jira-keyed branch and opens a PR for human review.

/minfyr:pr-reviewREVIEW

PR review

Runs the independent 9-category code reviewer on the PR — comment-only, never approves. Humans always merge.

/minfyr:cost-estimatorREPORT

Cost estimate

Projects AWS cost from architecture.md or a pasted brief — no AWS credentials required to estimate before committing.

/minfyr:security-scannerREPORT

Security scan

OWASP ASVS-L2 + AWS Well-Architected posture review before handover. Confirms CI gates are green. LLM Top 10 when AI surface is present.

Lean prototyping Internal tools & prototypes — one doc, scaffold, ship. No Jira, no Confluence, no ceremony.
/minfyr:internal-appLEAN

Internal app

Brief → one build-spec doc → scaffold + co-located single-env Terraform deploy. Frontend-only, full-stack, or backend (incl. AI). Graduate to the full pipeline any time.

Why teams choose minfyr

🔒

Design-as-contract

UX is locked into an OpenAPI contract code-generated for frontend and backend simultaneously — no API mismatch, ever.

🚀

Production-ready scaffolds

Every repo ships with CI/CD, OIDC to AWS (no long-lived keys), observability and ≥90% test coverage from day one.

📦

Feature-sliced delivery

The backlog is topologically sorted. Engineers ship one slice at a time with TDD and an independent 9-category code review.

🤖

AI-aware

When the architecture names LLM, agent or RAG components, minfyr generates prompt registries, agent inventories and eval suites automatically.

🔗

Jira + Confluence in sync

The backlog lives in Jira; specs, architecture and sign-offs live in Confluence — two-way sync keeps them aligned throughout delivery.

📊

Two delivery flows

Full customer pipeline (/minfyr:plan) or lean prototype (/minfyr:internal-app). Graduate a prototype to the full flow any time — nothing is thrown away.
